# 技术阳痿

<!-- DESC SEP -->

当“Hello World”的多巴胺褪去，我们剩下的是对无穷无尽框架的疲惫。是我们老了，还是技术本身变得无聊了？

<!-- DESC SEP -->

## 电子荒原上的贤者时间

最近在 V2EX 上看到一个[帖子](https://www.v2ex.com/t/1185349)，楼主即将入职，拥有大把空闲时间，却发现自己陷入了一种奇怪的状态：**技术阳痿**。

这不是职业倦怠（Burnout）。倦怠是过劳，是想逃离。而“技术阳痿”更像是一种欲望的丧失。

明明有时间，明明知道 AI Agent 是风口，明明知道 Web3 还在折腾，但面对那些花哨的名词，身体却诚实地毫无反应。打开 IDE 就像面对一个不再爱的前任，手指在键盘上悬停，最后还是打开了 Steam 或者 Netflix。

这种症状在工作 3-5 年的程序员中尤为普遍。我们不再像刚毕业时那样，看到一个新的 UI 库就两眼放光；不再为了配置一个 Vim 插件而熬夜通宵。

我们进入了职业生涯的“贤者时间”。

## 概念的通货膨胀

导致这种症状的第一个病因，是技术圈的**概念通货膨胀**。

十年前，学习 jQuery 是为了让网页动起来；学习 Java 是为了构建坚固的系统。那时候的技术，离“解决问题”很近。

而现在，我们被各种缩写包围：RAG、LLM、DDD、Serverless、Micro-frontends... 每一个新词都许诺一场革命。但当你剥开那层金光闪闪的包装纸，往往发现里面包裹的只是一个旧逻辑的蹩脚封装。

就像楼主在帖子里吐槽的：“感觉功能的提升，用户体验的提升，用不到多么复杂的 AI Agent 开发的技术。”

我们疲于奔命地追逐新框架，就像在跑步机上狂奔。我们学得越多，却感觉离真实的创造越远。这种**“为了学而学”的虚无感**，是扼杀技术热情的头号杀手。

当技术不再是手中的锤子，而变成了必须供奉的神像，我们自然会感到疲惫。

## 这种疲惫是保护机制

从进化的角度看，这种“阳痿”其实是一种心理保护机制。

大脑在告诉我们：**停下来，不要再吃垃圾食品了。**

那些为了 KPI 而造的轮子，那些为了显得高大上而引入的复杂架构，本质上就是认知层面的垃圾食品。它们只有热量，没有营养。

当你对“跟着教程写一个 Todo List”感到厌烦时，说明你的技术品味提升了。你不再满足于浅层的 API 调用，你开始渴望更深层的、更有逻辑美感的东西，或者是更纯粹的、能解决实际问题的能力。

这不叫阳痿，这叫**口味刁钻**。

## 重回具体

如何治愈？

评论区里有个老哥说得好：“没有需求瞎学个啥，大概率学了也没有。”

药方不在于更努力地刷 Github Trending，而在于**回归具体**。

不要去学“如何构建一个企业级高并发分布式 AI Agent 平台”。
去写一个脚本，帮你自动整理乱七八糟的下载文件夹。
去写一个插件，屏蔽掉浏览器里那些烦人的广告。
去帮你的朋友做一个简单的记账网页。

**伟大的代码往往诞生于微小的愤怒或具体的爱。**

当我们从宏大的叙事中抽离，回到具体的代码行，回到那个按下回车键、程序按照预期运行的瞬间，你会发现，那种久违的冲动又回来了。

技术不是目的，创造才是。

## 尾声

如果你现在正处于“技术阳痿”期，别慌。

这也许是生活在提醒你，该去锻炼身体了，该去读读《小狗钱钱》或者《反脆弱》了，或者仅仅是该去晒晒太阳了。

技术只是生活的一部分，而不是全部。当你不再执着于证明自己“跟得上时代”，你才真正拥有了技术的主权。

休息一下。等下一个真正值得解决的问题出现时，你会重新硬起来的。
